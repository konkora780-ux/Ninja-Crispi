# ニンジャクリスピー レシピ帳 — 引き継ぎ書

エアフライヤー **Ninja Crispi** 用のレシピ帳 Web アプリ。iPhone / iPad で使う。

- 公開先： https://konkora780-ux.github.io/Ninja-Crispi/
- 構成： **`index.html` 1枚だけ**。ライブラリ・ビルド・サーバーなし
- 保存： ブラウザの localStorage（キー `ninja-crispi-recipes-v1`、お手本投入フラグ `ninja-crispi-seeded-v2`）。端末ごとに独立、共有はしない

## 実機は日本モデル FN101J — ここを間違えない

- **温度設定はできない機種**。モードと時間だけを決める。アプリにも温度欄は無い
- モードは **クリスプ／ベイク・ロースト／リクリスプ ＋ 保温** の3＋1
- **マックスクリスプは日本モデルには無い**（海外の FN101GY 等にはある）。
  海外のレシピを取り込むときは **マックスクリスプ → クリスプ** に読み替える
- 予熱は不要。ガラスコンテナは L=3.8L / S=1.4L、耐熱230℃

## プレート（ノンスティックプレート）— レシピ必須項目

| 値 | 意味 |
|---|---|
| `あり` | プレートの上に食材をのせる。クリスプ・ベイクローストは基本これ。脂が下に落ちてカリッとする |
| `なし` | コンテナの底に直接入れる。しっとり温めたいとき。**ごはん・パスタのリクリスプは底に入れて、上からプレートをフタのようにかぶせる**（公式指定） |
| `途中から` | 下に野菜を入れて先に加熱 → あとからプレートを重ね、その上で肉を焼く二段調理 |

ゆるい生地（ホットケーキ等）をコンテナに直接流し込むのは禁止。プレート＋オーブンペーパーで焼く。

## レシピの形

```js
{ id, name, emoji, cat, mode, plate, min, size, serv,
  ing:[{n:'材料名', a:'分量'}], steps:['手順',…], memo, src, fav, seed }
```

- `seed:true` はお手本レシピ。⋯メニューの「入れ直す」で置きかえられる（ユーザー作成分は消えない）
- お手本36件の出典は NotebookLM のノートブック「Ninja Crispiレシピ」
  （id `2a5bc6ca-8ac6-465a-8ae1-09ee59f389b6`、ソース31件）。
  追加ネタは `nlm query notebook <id> "..."` で引ける（nlm は `C:\Users\konko\nlm-venv\Scripts\nlm.exe`）

## できあがり写真

- 写真は**レシピ本体とは別のキー** `ninja-crispi-photo-<レシピid>` に1枚ずつ保存する。
  レシピ本体の JSON に入れると、写真1枚で容量オーバーしたときにレシピ全体が保存できなくなるため
- 保存前に `shrinkImage()` で**長辺900pxのJPEG**へ縮小（品質0.72、420KB超ならさらに0.55）。
  2400x1600の写真で 138KB → 13KB
- 一覧を描くたびに localStorage を読まないよう `photoCache`（Map）を挟んでいる。
  `setPhoto` / `removePhoto` は必ずキャッシュも更新すること
- 書き出し（export）時だけ `photo` プロパティとして合流させ、読み込み時に別キーへ戻す
- localStorage 全体で5MBほどなので、写真は20〜30枚が上限の目安

## 公式レシピへのリンク（写真）

- **他社の写真はこのアプリに埋め込まない。** 公開URLで再配信することになり著作権上まずい。
  参照元（NotebookLM の googleusercontent など）の画像URLも一時的で切れる
- 代わりに**公式ページへのリンク**を持たせている。写真は公式の場所で見てもらう
- `OFFICIAL` 配列＝シャークニンジャ日本公式の Crispi 用レシピ106品（`[表示名, 'slug&type=NNNN']`）。
  ヘッダーの 📖 ボタンで一覧＋検索。URLは `OFFICIAL_BASE + slug&type`
  - 一覧の出どころ： https://www.sharkninja.jp/pages/ninjarecipes/?product=Crispi
    （**JSで描画されるので curl では3件しか取れない**。ブラウザで開いて
    `a[href*="recipe-detail"]` を拾うこと）
- レシピ側の `link` プロパティ … 同じ料理が公式にある18品に設定ずみ。詳細画面にボタンが出る。
  リンクは必ず `target="_blank" rel="noopener noreferrer"`

## 実装で気をつけている点

- `scaleAmount()`：分量の人数倍。日本語の分量は「大さじ1.5」「小さじ1/2」のように**数字が先頭に来ない**ので、
  文字列中の最初の数値だけを掛ける。`fmtNum()` が「2と1/4」のような分数表記に戻す。「少々」「適量」は素通り
- `migrate()`：旧バージョンのデータ（`temp` あり・`マックスクリスプ`）を今の形に直す。**消さないこと**
- タイマー：終了時刻(Date)からの逆算。iOS は裏に回ると `setInterval` が止まるため `visibilitychange` で描き直す
- アラーム音：WebAudio。iOS はユーザー操作なしに鳴らないので、タイマー開始のタップで `primeAudio()` して resume する
- iOS の ITP 対策で「ホーム画面に追加」を案内するバナーを出している
